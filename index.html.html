1. Introduction

HVM Panel is a powerful web-based control panel for managing Docker-based VPS instances. It provides a user-friendly interface for creating, managing, and monitoring virtual private servers with features like:

  • VPS creation and management
  • Resource monitoring
  • SSH access via web console
  • File management
  • User management
  • And much more

This guide will walk you through the complete installation and setup process.

2. Prerequisites

Before installing HVM Panel, ensure your system meets the following requirements:

1 System Requirements

  • Ubuntu 20.04/22.04 or CentOS 7/8 (recommended)
  • Minimum 2GB RAM (4GB+ recommended for multiple VPS instances)
  • At least 20GB of free disk space
  • Root or sudo access

2 Required Software

  • Python 3.8 or higher
  • Docker and Docker Compose
  • Git
Important: This panel creates and manages Docker containers. Make sure your system supports Docker and has sufficient resources for your planned VPS instances.

3. Installation Steps

1 Update System and Install Dependencies

First, update your system package repository and install basic dependencies:

# Update system packages
sudo apt update && sudo apt upgrade -y

# Install required dependencies
sudo apt install -y curl wget git python3 python3-pip python3-venv

For CentOS/RHEL systems:

# Update system packages
sudo yum update -y

# Install required dependencies
sudo yum install -y curl wget git python3 python3-pip

2 Install Docker

HVM Panel requires Docker to be installed. Run the following command to install Docker:

# Download and run Docker installation script
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh

# Add your user to the docker group
sudo usermod -aG docker $USER

# Start and enable Docker service
sudo systemctl start docker
sudo systemctl enable docker

# Verify Docker installation
docker --version
Note: After adding your user to the docker group, you may need to log out and log back in for the changes to take effect.

3 Download HVM Panel

Create a directory for the panel and download the source code:

# Create directory for HVM Panel
sudo mkdir -p /opt/hvm-panel
cd /opt/hvm-panel

# Download the panel code (replace with actual repository if available)
wget -q https://github.com/hvms/hvm/releases/download/V1/hvm.zip
# For this tutorial, we'll create the main Python file
sudo touch hvm_panel.py

Copy the HVM Panel Python code provided and save it as hvm_panel.py in the /opt/hvm-panel directory.

4 Install Python Dependencies

Create a Python virtual environment and install required packages:

# Create virtual environment
python3 -m venv venv

# Activate virtual environment
source venv/bin/activate

# Install required Python packages
pip install flask flask-socketio flask-login docker paramiko psutil python-dotenv email-validator

Alternatively, create a requirements.txt file with the following content:

flask
flask-socketio
flask-login
docker
paramiko
psutil
python-dotenv
email-validator
werkzeug

Then install all dependencies at once:

pip install -r requirements.txt

5 Create Necessary Directories

Create the required directories for the panel to function properly:

# Create directories for templates, uploads, and image cache
sudo mkdir -p templates static uploads image_cache

# Set proper permissions
sudo chown -R $USER:$USER /opt/hvm-panel
sudo chmod -R 755 /opt/hvm-panel

4. Configuration

1 Environment Configuration

Create a .env file to configure your panel settings:

# Create .env file
nano .env

Add the following configuration to the file (adjust values as needed):

# Panel Configuration
SECRET_KEY=your-secret-key-here
ADMIN_USERNAME=admin
ADMIN_PASSWORD=secure-admin-password
PANEL_NAME=HVM VPS Panel
WATERMARK=HVM VPS Service
WELCOME_MESSAGE=Welcome to HVM VPS Panel

# VPS Configuration
MAX_VPS_PER_USER=5
DEFAULT_OS_IMAGE=ubuntu:22.04
DOCKER_NETWORK=hvm_network
MAX_CONTAINERS=50

# Server Configuration
SERVER_IP=your-server-ip
SERVER_PORT=3000
DEBUG=False

# Email Configuration (Optional)
SMTP_SERVER=smtp.example.com
SMTP_PORT=587
[email protected]
SMTP_PASS=your-smtp-password
[email protected]
Security Note: Change the default admin credentials and generate a strong secret key before going live.

2 Create HTML Templates

The panel requires HTML templates to render the web interface. Create a templates directory and add the following basic templates:

# Create templates directory
mkdir -p templates

# Create basic login template
nano templates/login.html

Add this basic login template:

<!DOCTYPE html>
<html>
<head>
    <title>Login - {{ panel_name }}</title>
</head>
<body>
    <h1>Login to {{ panel_name }}</h1>
    {% if error %}
        <p style="color: red;">{{ error }}</p>
    {% endif %}
    <form method="POST">
        <input type="text" name="username" placeholder="Username" required>
        <input type="password" name="password" placeholder="Password" required>
        <button type="submit">Login</button>
    </form>
</body>
</html>

You'll need to create additional templates for dashboard, VPS management, etc. Refer to the Flask documentation for creating complete templates.

3 Create Systemd Service

Create a systemd service file to run the panel as a service:

# Create systemd service file
sudo nano /etc/systemd/system/hvm-panel.service

Add the following content to the service file:

[Unit]
Description=HVM VPS Panel
After=network.target docker.service

[Service]
Type=simple
User=root
WorkingDirectory=/opt/hvm-panel
Environment=PATH=/opt/hvm-panel/venv/bin
ExecStart=/opt/hvm-panel/venv/bin/python3 hvm_panel.py
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target

Enable and start the service:

# Reload systemd
sudo systemctl daemon-reload

# Enable the service to start on boot
sudo systemctl enable hvm-panel

# Start the service
sudo systemctl start hvm-panel

# Check service status
sudo systemctl status hvm-panel

5. Basic Usage

1 Accessing the Panel

Once the panel is running, access it through your web browser:

http://your-server-ip:3000

Use the admin credentials you set in the .env file to log in.

2 Creating Your First VPS

After logging in:

  1. Navigate to the "Create VPS" section
  2. Configure the VPS resources (CPU, RAM, Disk)
  3. Select an OS image
  4. Set expiration if needed
  5. Click "Create"

The panel will automatically build the Docker container and set up your VPS.

3 Managing VPS Instances

From the dashboard, you can:

  • Start/Stop/Restart VPS instances
  • Access VPS console via web SSH
  • Monitor resource usage
  • Manage files
  • Change passwords

6. Troubleshooting

Common Issues and Solutions

Issue: Panel fails to start
Solution: Check the service status and logs:
sudo systemctl status hvm-panel
sudo journalctl -u hvm-panel -f
Issue: Docker permission errors
Solution: Ensure your user is in the docker group and restart your session:
sudo usermod -aG docker $USER
Log out and log back in, or restart the system.
Issue: VPS creation fails
Solution: Check Docker daemon is running and has resources:
sudo systemctl status docker
docker info
Issue: Web interface not accessible
Solution: Check firewall settings and ensure port 3000 is open:
sudo ufw allow 3000 (for UFW)
Or check your cloud provider's firewall rules.

7. Security Considerations

Essential Security Practices

1. Change Default Credentials
Always change the default admin username and password in the .env file before deploying to production.
2. Use HTTPS
For production use, set up a reverse proxy with SSL/TLS encryption using Nginx or Apache.
3. Firewall Configuration
Restrict access to the panel port (3000) to trusted IP addresses only.
4. Regular Updates
Keep the system, Docker, and Python dependencies updated with security patches.
5. Monitor Resource Usage
Regularly monitor system resources and set up alerts for unusual activity that might indicate abuse.
